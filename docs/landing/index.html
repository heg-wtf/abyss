Built for power users

Everything runs locally. No cloud dependency.

🏠

Local First

No server, no cloud lock-in. Access via mobile PWA over Tailscale. Your data stays on your Mac.

🤖

Multi-Bot System

Run multiple specialized bots simultaneously. Each with its own personality, skills, model, and memory.

🧩

Skill Plugins

18+ built-in skills. Create custom ones with markdown or integrate MCP servers, CLIs, and browser tools.

⏰

Cron Automation

Schedule recurring tasks with natural language. Morning briefings, daily reports, periodic checks.

👥

Group Collaboration

Multi-bot orchestration with shared workspace — coming back on top of the mobile PWA.

🖥

Abysscope Dashboard

Web UI with frequency heatmap, cron editor, skill CRUD, session/conversation management, markdown memory viewer, and log tools.

⚡

SDK Client Pool

Persistent Claude sessions via Python Agent SDK. Fast responses with session reuse, no cold starts.

💓

Heartbeat

Periodic situation awareness. Bots proactively check tasks, reminders, and deliver updates autonomously.

📁

File-Based Architecture

No database. Sessions are directories. Conversations are markdown. Everything is inspectable and portable.

🔍

Conversation Search

Full-text search across all sessions via SQLite FTS5. Auto-injected as MCP tool. Rebuild anytime with abyss reindex.

🎙️

Voice Chat

Speak to your bot in the dashboard. ElevenLabs Scribe v2 STT, animated Orb visualization, TTS reply — full hands-free loop.

CLI First

Everything from onboarding to deployment, in the terminal. 

# Setup
$ abyss init
$ abyss bot add

# Run — API + dashboard + schedulers in one
$ abyss start
$ abyss status

# Skills
$ abyss skills install supabase
$ abyss skills install reminders

# Automation
$ abyss cron add my-bot
$ abyss heartbeat enable my-bot

What's New

v2026.05.15 One process, fewer dials — `abyss start` now boots the API server, the dashboard subprocess, and the per-bot schedulers as a single Rich `BuildProgress` checklist; the standalone `abyss dashboard` subcommand and the `--daemon` flag are retired (foreground is the default so the boot checklist is visible; `--daemon` for background). The OpenAI-compatible backends (`openai_compat` / `openrouter` / `minimax`) are removed too — abyss is a Claude Code persona agent toolkit, the simpler surface is the point. Mobile chat picks up single-slot message queueing (Send while a reply is still streaming → it fires the moment the turn finishes), a cleaner three-dot streaming indicator with an inline ✕ cancel, and a new Settings dialog (gear icon in the sidebar) hosting Light / Dark / System theme — first of more quick toggles to come. Logo / favicon / PWA icon pipeline now flows through a square 600 × 600 source so home-screen tiles and apple-touch render without letterboxing.
v2026.05.14 Mobile PWA as primary surface — Bot messaging and group collaboration removed. The mobile PWA ( /mobile on the abysscope dashboard) is now the primary user-facing interface, accessible over Tailscale. Cron and heartbeat results land in the Routines tab and are delivered via Web Push notifications. Dashboard chat (abysscope) continues to work for desktop. Multi-bot group collaboration will return on top of the PWA in a future release.
v2026.05.13 Built-in skills slimmed — bundled set now covers universal tooling only (email, calendar, OS integrations, generic utilities). Removed 6 Korea-specific domain skills; author your own or import via abyss skills import <github-url>. Skill Authoring Guide — docs/SKILL_AUTHORING.md walks through directory layout, skill.yaml fields, GitHub sharing, and the untrusted-skill security guard. Per-session chat stream isolation — dashboard chat SSE streams no longer cross tokens between concurrent bot sessions. Repo cleanup — completed plan docs dropped; dashboard's built-in skill set now correctly includes code_review and conversation_search.
v2026.05.08 Streaming: typing indicator — shows typing… immediately on message receipt, cancelled when the first draft chunk arrives (covers Claude cold-start gap). Tool metrics rotation fix — _rotate() now uses the event timestamp instead of wall clock, preventing test-mode events from being pruned immediately.
v2026.05.05 Voice Chat — mic button in the dashboard chat opens a right sidebar with an animated ElevenLabs Orb. ElevenLabs Scribe v2 WebSocket STT transcribes speech in real time; transcript is sent to the bot with a spoken-style instruction so replies sound natural. TTS streaming plays the bot reply via Web Audio API. Orb color is theme-aware (white on dark, black on light). Auto-restart loop keeps the mic hot after each reply. Messenger-style timestamps on all chat messages.
v2026.05.03 Dashboard Chat — top-level /chat page with bot picker, streaming SSE responses, and per-session avatars. Sessions live in chat_web_* under the bot's workspace. Image + PDF attachments via paperclip / drag & drop / clipboard paste — files land in workspace/uploads/ and Claude opens them through its Read tool. Magic-byte MIME sniff, per-session upload cap, path-traversal guard. New build progress UI for abyss restart: replaces the raw next build firehose with a 4-step Rich checklist, captures the build log under ~/.abyss/logs/dashboard-build-*.log, surfaces failures with the last 30 lines. Collapsible sidebar with persisted state. Test coverage 69 → 71% (+52 cases).
v2026.04.30 Claude Code 2.1.x Integration — six-phase hardening + telemetry roll-up. 1-hour prompt cache, forked subagents in non-interactive mode, MCP non-blocking startup + alwaysLoad, hidden cwd. PreCompact hook drives MEMORY/HEARTBEAT compaction from Claude Code's own pressure signal. PostToolUse + PostToolUseFailure hooks record per-tool latency to JSONL — new Metrics tab in abysscope shows p50/p95/p99 per tool. Skills can declare conditional hooks via if field. GitHub-imported skills run with disableSkillShellExecution; cloud-metadata endpoints blocked by default via sandbox.network.deniedDomains. New run_ultrareview wrapper + code_review builtin skill, per-bot effort level. Avatar fetch resilience (try/catch + stale cache fallback + IPv4 first).
v2026.04.29 Frequency Heatmap — GitHub contribution graph-style activity heatmap on the dashboard. Shows daily conversation count across all bots with per-bot tooltip breakdown. Bot profile photos. Cleaner UI (removed stat cards). Dashboard memory ~200 MB (was 4 GB).
v2026.04.24 Multi-Provider LLM — openai_compat backend for MiniMax, OpenRouter, or any OpenAI-compatible endpoint. Per-bot opt-in via bot.yaml.
v2026.04.26 Abysscope Search — Full-text conversation search tab in the web dashboard, powered by SQLite FTS5.
v2026.04.25 Conversation Search — Cross-session FTS5 search auto-injected as search_conversations MCP tool. Rebuild index with abyss reindex.
v2026.03.25 Bot Isolation — Each bot's CLAUDE.md now explicitly ignores parent configs (~/.claude/CLAUDE.md). Fixed dashboard PID OverflowError crash.
v2026.03.21 Abysscope v2 — Cron editor (recurring/one-shot), custom skill CRUD, session & conversation delete, markdown memory viewer, /skills bot-scoped view.
v2026.03.20 Cron Edit — Edit cron job messages via CLI (abyss cron edit with $EDITOR) or mobile + dashboard (/cron edit). Expanded cron list preview to 80 chars.
v2026.03.18 GitHub Skill Import — Import skills directly from GitHub repos via CLI (abyss skills import) or via abysscope dashboard / mobile chat. Compatible with skills.sh packages.
v2026.03.16 SDK Client Pool — Persistent Claude sessions via Python Agent SDK. Removed Node.js bridge dependency.
v2026.03.14 Group Collaboration — Multi-bot group chat with shared conversations, workspace, and orchestrator routing.
v2026.03.12 Abysscope Dashboard — Web UI for managing bots, skills, cron, sessions, memory, and logs.
v2026.03.11 Dashboard CLI — Added start/stop/restart/status subcommands for Abysscope management.
